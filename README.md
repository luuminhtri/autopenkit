AutoPenKit

AutoPenKit is a Python CLI-based automated pentesting framework with AI-assisted analysis.

Purpose

This project helps automate parts of an authorized security testing workflow:

* Target validation
* Reconnaissance
* Vulnerability scanning
* Result normalization
* AI-assisted analysis
* Report generation

Legal Notice

This tool is intended for educational and authorized security testing only.

Allowed use cases:

* Local labs
* OWASP Juice Shop
* DVWA
* Intentionally vulnerable applications
* Targets with explicit permission

Do not use this tool on unauthorized systems.

Phase 1 Goals

In Phase 1, the project should be able to:

* Run from the command line
* Validate a target URL
* Check whether the target is authorized
* Create a unique output folder for each scan
* Generate basic metadata
* Generate an initial asset list

Expected command:

python main.py –target http://localhost:3000 –profile safe

Expected output:

outputs/<scan_id>/
├── validated_target.json
├── scan_metadata.json
└── assets.json

Safety Principle

AutoPenKit is designed with safety-first defaults.

By default, only local lab targets should be allowed, such as:

* localhost
* 127.0.0.1

Other targets must be explicitly added to the authorized scope in the configuration file.
